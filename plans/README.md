## Findings Considered And Rejected

- Plaintext `roomId`, `seq`, and `senderId` routing metadata: accepted current spike tradeoff documented in `PLAN.md` and `AGENTS.md`, not a bug for this plan set.
- Lack of fork/truncation detection, signed checkpoints, compaction, key rotation, and production durability: important but already explicitly documented as open hardening work; not treated as newly discovered regressions here.
- Source-only edit mode: current repo instructions say edit mode is source-only for now despite the broader product roadmap discussing richer editing, so this plan set does not reintroduce editor mode churn.
- CORS wildcard on the append-log server: notable for deployment hardening, but current no-cookie/no-server-secret append-log shape makes it lower leverage than the browser CSP, local key persistence, and secret output items.
